# [LeetCode 64] 四种思路，三种可行

首先看到这个题，我的第一反应是DFS + 回溯，反正每次就往下或者往右，遇到障碍物就回溯呗。所以我写了这样的代码：

function uniquePathsWithObstacles(obstacleGrid: number[][]): number {
  const m = obstacleGrid.length,
    n = obstacleGrid[0].length;
  let paths = 0;
  function traverse(i: number, j: number) {
    if (obstacleGrid[i][j] === 1) return;
    if (i === m - 1 && j === n - 1) ++paths;
    if (i + 1 < m) traverse(i + 1, j);
    if (j + 1 < n) traverse(i, j + 1);
  }
  traverse(0, 0);
  return paths;
}

但是这样会超时，因为重复计算的次数太多了。

仔细想想，到达每一个格子的方法数，完全取决于到达它上面一格和左边一格的方法数，是一个标准的dp问题。所以，也可以很自然地写出这样的dp代码：

function uniquePathsWithObstacles(obstacleGrid: number[][]): number {
  if (obstacleGrid[0][0] == 1) return 0;
  const m = obstacleGrid.length,
    n = obstacleGrid[0].length;
  const dp = new Array(m);
  for (let i = 0; i < m; ++i) dp[i] = new Array(n);
  dp[0][0] = 1;
  for (let i = 1; i < m; ++i)
    dp[i][0] = obstacleGrid[i][0] === 1 || dp[i - 1][0] === 0 ? 0 : 1;
  for (let j = 1; j < n; ++j)
    dp[0][j] = obstacleGrid[0][j] == 1 || dp[0][j - 1] == 0 ? 0 : 1;
  for (let i = 1; i < m; ++i) {
    for (let j = 1; j < n; ++j) {
      dp[i][j] = obstacleGrid[i][j] == 1 ? 0 : dp[i - 1][j] + dp[i][j - 1];
    }
  }
  return dp[m - 1][n - 1];
}

但是这个的边界情况比较麻烦：

  • 首先，起点(0, 0)可能存在障碍物，这样就不可能到达终点，方法数为0；
  • 第一列不存在上一列，如果前一格无法到达，或者这一格本身就有障碍，那么这一格是无法到达的，方法数为0；而如果能够到达，也只可能是从起点一直直走，只有这一种方法，所以第一列只有0和1。
  • 第一行同理，也只有0和1。

然后就是想着能不能对dp占用的空间做优化，m * n的空间还是太大了。一般来说，对dp占用的空间做优化，可能有两种思路：

  • 原地dp（不是原地tp），在允许修改原数据的情况下，尝试使用原数据进行dp；
  • 滚动数组，复用之前使用的dp空间。

先说说在原数组上进行dp。用原数组进行dp有一个问题，就是到达某一格的方法数可能为1，这就会和原始的障碍物（也表示为1）产生混淆。所以，我们不妨用负数来表示方法数，这样就不会与原数组的内容冲突了，然后在取结果的时候乘上-1即可。当然也有别的表示法，我只是给一个可能的办法：

function uniquePathsWithObstacles(obstacleGrid: number[][]): number {
  if (obstacleGrid[0][0] == 1) return 0;
  const m = obstacleGrid.length,
    n = obstacleGrid[0].length;
  obstacleGrid[0][0] = -1;
  for (let i = 1; i < m; ++i)
    obstacleGrid[i][0] =
      obstacleGrid[i][0] === 1 || obstacleGrid[i - 1][0] === 0 ? 0 : -1;
  for (let j = 1; j < n; ++j)
    obstacleGrid[0][j] =
      obstacleGrid[0][j] === 1 || obstacleGrid[0][j - 1] === 0 ? 0 : -1;
  for (let i = 1; i < m; ++i) {
    for (let j = 1; j < n; ++j) {
      obstacleGrid[i][j] =
        obstacleGrid[i][j] === 1
          ? 0
          : obstacleGrid[i - 1][j] + obstacleGrid[i][j - 1];
    }
  }
  return -obstacleGrid[m - 1][n - 1];
}

然后就是滚动数组，这个的基本思想就是复用之前的dp空间。我们注意到，我们计算到达每一格的方法数，只会关心和它直接相邻的上面一格和左边一格。但是只用两个变量去存储上面一格和左边一格是不行的，因为要在某一边不变的情况下移动到另一边的后一格，只用两个变量没法做到这一点。

可以想象一下，两个变量相当于两个点，两个点在平面上移动只能模拟出一条线，而一个数组（相当于一条线）在平面上移动，则可以模拟出一个面。所以，可以考虑用一个数组去存放某一行（或者某一列）上的数据：

function uniquePathsWithObstacles(obstacleGrid: number[][]): number {
  if (obstacleGrid[0][0] == 1) return 0;
  const m = obstacleGrid.length,
    n = obstacleGrid[0].length;
  const lastCol = new Array(n).fill(1);
  for (let j = 1; j < n; ++j)
    lastCol[j] = obstacleGrid[0][j] === 1 || lastCol[j - 1] === 0 ? 0 : 1;
  for (let i = 1; i < m; ++i) {
    lastCol[0] = obstacleGrid[i][0] === 1 || lastCol[0] === 0 ? 0 : 1;
    for (let j = 1; j < n; ++j) {
      lastCol[j] = obstacleGrid[i][j] === 1 ? 0 : lastCol[j - 1] + lastCol[j];
    }
  }
  return lastCol[n - 1];
}
