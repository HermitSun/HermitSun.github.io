# [LeetCode 1014] dp与滚动数组优化

首先第一反应估计是暴力法，但是暴力法会超时。但是为什么会超时呢，之前看到一组数据，说是1s执行的计算量大概是3*10^8，也就是说，如果算法复杂度为O(n^2)，能在1s内完成的任务规模大概在10,000左右。而我们可以看到，这一题是2 <= A.length <= 50000，显然是会超时的。

而事实上，A[i] + A[j] + i - j = (A[i] + i) + (A[j] - j)，只要让两者同时最大即可。而对于固定的j来说，A[j] + j是个定值，所以只需要让另一项最大即可。而i < j，这问题就变成了，找到当前项之前的项中让A[i] + i最大的那个值；可以用一个数组来存储一路上遇到的A[i] + i的最大值，然后不断更新，这就是dp的思路了。

function maxScoreSightseeingPair(A: number[]): number {
  const dp = [A[0]], length = A.length;
  let maxScore = 0;
  for (let i = 1; i < length; ++i) {
    maxScore = Math.max(maxScore, A[i] - i + dp[i - 1]);
    dp[i] = Math.max(dp[i - 1], A[i] + i);
  }
  return maxScore;
}

但是仔细看看就会发现，这个dp数组是没什么意义的，因为我们每次只依赖于前一个状态。所以，就可以借鉴滚动数组的思想，只用一个变量来存储上一个状态，避免过多的内存开销。

function maxScoreSightseeingPair(A: number[]): number {
  const length = A.length;
  let maxScore = 0, dp = A[0];
  for (let i = 1; i < length; ++i) {
    maxScore = Math.max(maxScore, A[i] - i + dp);
    dp = Math.max(dp, A[i] + i);
  }
  return maxScore;
}

这题的关键点主要还是把式子拆开。
